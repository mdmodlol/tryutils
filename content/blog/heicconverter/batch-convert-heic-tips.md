## 方法二：使用脚本自动化

### Windows PowerShell 脚本

适合技术用户，可以完全自动化处理：

```powershell
# 批量转换 HEIC 到 JPG 的 PowerShell 脚本
$sourceFolder = "C:\Photos\HEIC"
$outputFolder = "C:\Photos\JPG"

# 确保输出文件夹存在
if (!(Test-Path $outputFolder)) {
    New-Item -ItemType Directory -Path $outputFolder
}

# 获取所有 HEIC 文件
$heicFiles = Get-ChildItem -Path $sourceFolder -Filter "*.heic" -Recurse

foreach ($file in $heicFiles) {
    $outputFile = Join-Path $outputFolder ($file.BaseName + ".jpg")
    # 这里需要调用转换工具
    Write-Host "转换: $($file.Name) -> $($outputFile)"
}
```

### macOS 自动化脚本

```bash
#!/bin/bash
# 批量转换 HEIC 到 JPG

SOURCE_DIR="/Users/username/Pictures/HEIC"
OUTPUT_DIR="/Users/username/Pictures/JPG"

# 创建输出目录
mkdir -p "$OUTPUT_DIR"

# 批量转换
for file in "$SOURCE_DIR"/*.heic; do
    filename=$(basename "$file" .heic)
    sips -s format jpeg "$file" --out "$OUTPUT_DIR/$filename.jpg"
done
```

#### 📁 文件整理

## 常见问题和解决方案

### Q1: 批量转换时部分文件失败怎么办？

**可能原因：**
- 文件损坏
- 文件名包含特殊字符
- 磁盘空间不足
- 权限问题

**解决方案：**
1. 检查失败文件是否能单独打开
2. 重命名包含特殊字符的文件
3. 清理磁盘空间
4. 以管理员权限运行转换工具

### Q2: 转换后照片的拍摄时间丢失了？

**解决方案：**
- 使用支持 EXIF 数据保留的转换工具
- TryUtils 默认保留所有元数据
- 避免使用简单的图片查看器进行转换

### Q3: 批量转换速度太慢？

**优化建议：**
- 关闭其他占用 CPU 的程序
- 使用 SSD 硬盘存储临时文件
- 选择合适的质量设置（不要过高）
- 考虑分批处理

### Q4: 转换后文件大小差异很大？

**正常现象：**
- 复杂场景（细节多）→ 文件大
- 简单场景（天空、纯色）→ 文件小
- 这是 JPG 压缩算法的特性
