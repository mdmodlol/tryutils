# TryUtils 指标规则（Business Harness）

## 指标设计原则

- 只看能直接帮助判断“该做什么”的指标
- 指标数量宁少勿多
- 不追求大而全的数据看板
- 当前以 GSC 手动导出 CSV 为唯一稳定来源

## 当前唯一关注的业务指标

### 1. 主线博客点击

定义：

- `网页.csv` 中 URL 包含 `/blog/imagecompression/` 或 `/blog/heicconverter/`
- 点击次数总和

意义：

- 说明主线内容有没有实际拿到搜索点击

### 2. 主线博客展示

定义：

- `网页.csv` 中主线博客 URL 的展示总和

意义：

- 说明 Google 是否正在给主线内容分发曝光

### 3. 主线页面 CTR

定义：

- 主线博客页与主线工具页各自的点击率
- CTR = 点击次数 / 展示

意义：

- 说明标题、描述、页面意图匹配是否够好

### 4. 主线页面平均排名

定义：

- `网页.csv` 或 `查询数.csv` 中与主线相关页面/主题的平均排名

意义：

- 说明当前最值得做的是提排名，还是提 CTR

### 5. 博客到工具点击率

定义：

- 当前仓库内无法直接从 GSC 得到
- 先通过页面结构和后续站内事件埋点预留
- 在没有埋点前，作为“需要补充的数据能力”记录

意义：

- 这是主线内容是否真的能导工具的核心指标

### 6. 工具页自然搜索点击

定义：

- `网页.csv` 中主线工具页 URL 的点击总和

意义：

- 说明工具页是否已经开始具备独立搜索价值

### 7. 高展示低 CTR 页面清单

定义：

- 展示高于当前快照平均水平
- CTR 明显偏低
- 以主线页面优先

意义：

- 这是当前最该改标题、描述、首段和页面承接的页面池

## 从 GSC CSV 派生指标的规则

### 数据文件

默认使用：

- `data/gsc-snapshots/YYYY-MM-DD/查询数.csv`
- `国家_地区.csv`
- `设备.csv`
- `图表.csv`
- `网页.csv`

其中最核心的是：

- `查询数.csv`
- `网页.csv`
- `设备.csv`
- `国家_地区.csv`

### 主线识别规则

默认按以下关键词或路径识别：

- compression：`imagecompression`、`image-compressor`、`compress`
- heic：`heicconverter`、`heic-converter`、`heic`

### 主题归类规则

GSC summary 中至少归类为：

- `compression`
- `heic`
- `other`

## 增长信号

下面这些现象默认算增长信号：

- 主线页面展示持续上升
- 主线页面进入前 20，尤其是前 10
- 某篇主线文章 CTR 明显改善
- 工具页自然点击开始出现或稳定增长
- 主线博客的高价值主题越来越集中

## 异常信号

下面这些现象默认算异常信号：

- 主线展示增长但点击长期不涨
- 高展示页面 CTR 长期过低
- 主线页面排名明显下滑
- 工具页自然点击长期接近 0
- 非主线页面消耗大量精力，但对主线指标没有帮助

## 看到指标后怎么判断下一步

### 情况 A：展示高、CTR 低

优先做：

- 改标题
- 改 meta description
- 改首段
- 改 CTA

### 情况 B：排名在 5-20

优先做：

- 加强页面意图匹配
- 补 FAQ
- 强化内容结构
- 优化内链和 related tool

### 情况 C：博客页有点击、工具页没有点击

优先做：

- 优化博客到工具页的承接
- 改 CTA 位置和文案
- 改工具页场景文案

### 情况 D：非主线页面有少量流量

默认不追

除非：

- 能明显反哺主线
- 或已经形成稳定流量资产且维护成本极低
