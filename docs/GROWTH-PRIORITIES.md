# TryUtils 增长优先级（Business Harness）

## 唯一业务定位

- TryUtils 不是泛在线工具站。
- TryUtils 是 **隐私优先的浏览器端图片处理站**。
- 核心价值不是“工具很多”，而是“让用户在浏览器里快速完成图片压缩、HEIC 转换和格式处理”。

## 唯一主线

- `image-compressor`
- `heic-converter`
- `image-format-converter`

这三条主线之外的老工具：

- 保留访问入口
- 允许修 bug
- 不再主动增加 SEO、内容、首页露出和功能投入

## 当前月唯一重点

- 用 GSC 数据驱动主线页面增长
- 先把已有高展示页面的点击率和导流能力做起来
- 让博客页更稳定地把搜索流量导向主线工具页

## 明确不做的事

- 不再扩回“泛工具站”路线
- 不以 Google AdSense 作为当前主增长目标
- 不为非主线工具新增内容集群
- 不为了“看起来忙”而做低价值 UI 微调
- 不在没有 GSC 证据的情况下批量写文章
- 不把业务判断外包给临时聊天记录或口头说明

## 主线优先顺序

默认优先顺序固定如下：

1. 高展示且已经进入前 3 页的主线页面
2. 能把博客流量导向工具页的内容联动与 CTA
3. `imagecompression` 与 `heicconverter` 的内容维护
4. `image-compressor` 与 `heic-converter` 的关键体验优化
5. `image-format-converter` 作为主线辅助工具的承接优化
6. 非主线老工具的低成本维护

## 页面优先级判断规则

Claude 或其他 agent 做业务判断时，按这个顺序筛选页面：

1. `主线页面 + 高展示 + CTR 低`
2. `主线页面 + 排名 5-20 + 有点击或明显展示`
3. `主线博客 + 已有流量 + CTA 弱`
4. `工具页 + 自然点击开始增长 + 体验明显不顺`
5. `非主线页面`

如果两个页面都值得做，优先选：

- 更接近主线的
- 更接近前 10 名的
- 更容易通过 CTA 导到工具页的

## 内容与功能的取舍规则

默认优先做：

- 改已有高价值文章的标题、描述、首段、CTA、FAQ
- 改已有主线工具页的场景承接、转化文案和任务完成体验

默认暂缓做：

- 没有展示数据支撑的新主题
- 没有主线价值的开发者工具内容扩张
- 只提升“好看”但不提升转化的设计改动

## Agent 执行时的固定判断规则

当 Claude 接到“自己决定下一步做什么”这类任务时，必须先读：

- `docs/session-state.md`
- `docs/GROWTH-PRIORITIES.md`
- `docs/CONTENT-OPS.md`
- `docs/METRICS.md`
- `docs/WEEKLY-REVIEW.md`
- 最新的 `docs/analytics/latest-gsc-summary.md`

然后按下面顺序判断：

1. 当前月唯一重点是什么
2. 最近一份 GSC summary 里最高价值的问题是什么
3. 是该先优化页面，还是先优化工具体验
4. 本周明确不做什么

## 允许做的高价值动作

- 优化主线文章的 CTR
- 优化主线博客的 CTA 与 related tool
- 优化主线工具页的场景预设、目标结果和承接文案
- 补业务规则文档、复盘模板、数据目录和轻脚本

## 禁止做的低价值动作

- 批量新增非主线内容
- 新做一个没有 GSC 证据支撑的工具
- 仅凭“感觉可能有流量”扩方向
- 不看 weekly review 就直接决定大方向
