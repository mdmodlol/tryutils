# 可访问性测试指南

## 键盘导航测试

### 基本键盘导航测试步骤

1. **Tab 键导航测试**
   - 使用 Tab 键在页面元素间导航
   - 确保焦点顺序符合逻辑（从左到右，从上到下）
   - 检查所有可交互元素都能通过 Tab 键访问
   - 使用 Shift+Tab 进行反向导航

2. **跳转到主内容链接测试**
   - 页面加载后按 Tab 键
   - 第一个焦点应该是"跳转到主内容"链接
   - 按 Enter 键应该跳转到主内容区域

3. **语言切换器测试**
   - Tab 到语言切换按钮
   - 按 Enter 或 Space 键打开下拉菜单
   - 使用 Arrow Up/Down 键在选项间导航
   - 按 Enter 选择语言
   - 按 Escape 键关闭菜单

4. **导航菜单测试**
   - Tab 到导航菜单项
   - 检查每个菜单项都能获得焦点
   - 按 Enter 键应该能导航到相应页面

5. **表单控件测试**
   - Tab 到文件上传区域
   - 按 Enter 或 Space 键应该打开文件选择对话框
   - 检查所有按钮都能通过键盘激活

### 页面特定测试

#### 博客页面测试

- 检查标签过滤器的键盘导航
- 测试文章卡片的键盘访问

#### 博客文章页面测试
- 测试面包屑导航的键盘操作
- 检查分享按钮的键盘访问

## 屏幕阅读器测试

### 推荐的屏幕阅读器
- **Windows**: NVDA (免费) 或 JAWS
- **macOS**: VoiceOver (内置)
- **Linux**: Orca

### 屏幕阅读器测试步骤

1. **页面结构测试**
   - 使用标题导航 (H1-H6)
   - 检查标题层级是否合理
   - 测试地标导航 (banner, main, navigation, contentinfo)

2. **内容朗读测试**
   - 检查所有文本内容都能被正确朗读
   - 验证图片的 alt 文本是否描述准确
   - 测试链接文本是否有意义

3. **表单标签测试**
   - 检查所有表单控件都有适当的标签
   - 验证错误消息能被屏幕阅读器识别
   - 测试必填字段的标识

4. **交互元素测试**
   - 测试按钮的角色和状态公告
   - 检查下拉菜单的展开/收起状态
   - 验证模态对话框的焦点管理

### ARIA 属性测试

1. **aria-label 和 aria-labelledby**
   - 检查复杂控件的标签
   - 验证图标按钮的描述

2. **aria-describedby**
   - 测试帮助文本的关联
   - 检查错误消息的关联

3. **aria-expanded**
   - 测试下拉菜单的状态公告
   - 检查折叠/展开控件

4. **aria-live**
   - 测试动态内容更新的公告
   - 检查状态消息的实时更新

## 自动化测试工具

### 浏览器扩展
1. **axe DevTools** (推荐)
   - 安装 Chrome/Firefox 扩展
   - 在开发者工具中运行 axe 扫描
   - 查看并修复发现的问题

2. **WAVE Web Accessibility Evaluator**
   - 安装浏览器扩展
   - 运行页面扫描
   - 查看可访问性问题报告

3. **Lighthouse Accessibility Audit**
   - 在 Chrome DevTools 中运行 Lighthouse
   - 查看可访问性评分和建议

## 常见问题和解决方案

### 焦点管理问题
- **问题**: 焦点跳过某些元素
- **解决**: 检查 tabindex 设置，确保可交互元素有正确的 tabindex

### 屏幕阅读器公告问题
- **问题**: 动态内容更新不被公告
- **解决**: 使用 aria-live 属性标记动态区域

### 键盘陷阱问题
- **问题**: 焦点被困在某个组件中
- **解决**: 实现正确的键盘事件处理，确保 Escape 键能退出

### 语义化问题
- **问题**: 屏幕阅读器无法理解页面结构
- **解决**: 使用正确的 HTML 语义元素和 ARIA 角色
